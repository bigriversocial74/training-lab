# Mobile + Accessibility Completion v1

## Outcome

Training Lab now provides a shared accessibility and mobile-completion layer across public, participant, reviewer, merchant, and administrator surfaces.

## Included

- skip-to-main-content support;
- visible `:focus-visible` indicators;
- 44px minimum interactive controls;
- keyboard-contained mobile navigation drawers with focus return;
- Escape-key close behavior;
- screen-reader announcements for drawer state and product status;
- automatic focus on form error summaries;
- reduced-motion and forced-colors support;
- responsive reflow for product heroes, statistics, cards, actions, and forms;
- horizontal containment for wide tables and code;
- an accessibility statement and support route;
- removal of obsolete local-storage demo state from the global application script.

## Zoom and viewport

The viewport does not disable zoom and does not set a restrictive maximum scale. Browser text scaling and page zoom remain available.

## Authority boundary

Accessibility helpers change presentation, focus, and announcement behavior only. They do not create accounts, roles, campaigns, proof, reviews, rewards, claims, payments, wallets, redemptions, or Microgifter authority.

## SQL

No SQL is required.
