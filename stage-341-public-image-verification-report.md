# Stage 341 Public Image Verification + Template Correction

## Reason for pass
Stage 340 integrated every uploaded image into the app, but the public-page pass still needed semantic verification. In particular, the blog index could still present non-blog/icon-style assets as article visuals and a few public hero/CTA placements still used generic fallback art where a page-specific PNG existed.

## Audit findings

Initial score: **8.3 / 10**

Issues found:
- Blog cards were generated by the generic card renderer, so non-icon artwork could be squeezed into icon/card slots instead of full article-image slots.
- The second and third blog posts did not use intentional full-width article artwork.
- The home/about/success CTA or hero placements still used the generic `training_lab_hero` fallback in places where page-specific artwork was available.
- The design readiness API verified image presence and usage mapping, but not whether public-page major image slots were using the right class of assets.

## Fixes applied

- Rebuilt `blog.php` as a dedicated blog landing layout.
- Kept the first blog post using `blog_article` as the main image.
- Replaced the second blog post visual with `hero_task_reward` as a full-width article image.
- Replaced the third blog post visual with `receipt_visual` as a full-width article image.
- Added public template logic so marketing/app/admin artwork renders as media cards, while icons remain small icon cards.
- Replaced public hero/CTA fallback usage with page-specific artwork where appropriate.
- Added `tl_design_public_image_audit()` to verify public major image slots.
- Added public-image audit output to `/api/training/design-assets.php` and ops design summary.
- Updated CSS for media cards and verified blog cards.
